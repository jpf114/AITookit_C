# 安全策略

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 1.2.x | ✅ 当前维护 |
| < 1.2 | ❌ 不再维护 |

## 报告漏洞

如果你发现 AIToolkit_C 的安全问题，请**不要**在公开 Issue 中讨论。

推荐方式：

1. 在 GitHub 仓库使用 [Security Advisories](https://github.com/jpf114/AITookit_C/security/advisories/new) 提交私有报告；或
2. 通过仓库维护者提供的私有联系渠道说明问题（若已配置）。

请在报告中尽量包含：

- 受影响版本
- 复现步骤或 PoC
- 影响范围（本地提权、远程代码执行、数据泄露等）
- 建议修复思路（如有）

## 响应时间

- **确认收到**：3 个工作日内
- **初步评估**：7 个工作日内
- **修复发布**：视严重程度，通常 30 天内提供补丁版本

## 范围说明

本策略涵盖：

- 应用程序本地行为（文件读写、路径处理、崩溃处理）
- 模型与 catalog 下载链路（URL 校验、完整性校验）
- 依赖链中可由本项目缓解的风险

不在范围内：

- 第三方模型本身的训练偏见或误判
- 用户自行导入的不可信 ONNX 模型
- 操作系统或 Qt / ONNX Runtime 上游漏洞（会跟踪上游补丁并发布新版本）

## 安全相关配置

模型目录 URL 白名单、SHA256 校验等策略见 [docs/CONFIGURATION.md](docs/CONFIGURATION.md)。
