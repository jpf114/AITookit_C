# 模型配置 / 模型清单规范

## 功能说明

模型清单（Model Manifest）是描述 ONNX 模型元信息的 JSON 配置文件，包含模型名称、任务类型、推理后端、输入尺寸、阈值参数和类别标签等。本工具通过解析模型清单来加载和配置推理模型。

## 文件格式

模型清单为 JSON 文件，与 ONNX 模型文件放在同一目录下，文件名通常为 `{model_base_name}.json`。

### 示例

#### 检测模型

```json
{
  "name": "YOLOv8n COCO",
  "task_type": "detection",
  "backend": "onnxruntime",
  "model": "yolov8n.onnx",
  "input_width": 640,
  "input_height": 640,
  "confidence_threshold": 0.25,
  "nms_threshold": 0.45,
  "labels_inline": ["person", "bicycle", "car", ...]
}
```

#### 分类模型

```json
{
  "name": "YOLOv8n-cls ImageNet",
  "task_type": "classification",
  "backend": "onnxruntime",
  "model": "yolov8n-cls.onnx",
  "input_width": 224,
  "input_height": 224,
  "labels_inline": ["tench", "goldfish", ...]
}
```

#### 分割模型

```json
{
  "name": "YOLOv8n-seg COCO",
  "task_type": "segmentation",
  "backend": "onnxruntime",
  "model": "yolov8n-seg.onnx",
  "input_width": 640,
  "input_height": 640,
  "confidence_threshold": 0.25,
  "nms_threshold": 0.45,
  "labels_inline": ["person", "bicycle", "car", ...]
}
```

## 字段说明

### 必填字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| name | string | 模型名称，不能为空 |
| task_type | string | 任务类型：`detection`（检测）、`classification`（分类）、`segmentation`（分割） |
| backend | string | 推理后端，当前仅支持 `onnxruntime` |
| model | string | ONNX 模型文件路径，相对于清单文件，必须存在 |
| input_width | int | 模型输入宽度，必须为正整数 |
| input_height | int | 模型输入高度，必须为正整数 |

### 可选字段

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| confidence_threshold | double | `0.25` | 置信度过滤阈值，范围 0.0–1.0 |
| nms_threshold | double | `0.45` | NMS IoU 阈值，范围 0.0–1.0（仅 detection/segmentation 有效） |
| labels | string | — | 标签文件路径，相对于清单文件 |
| labels_inline | string[] | `[]` | 内联标签列表，优先于 `labels` 文件 |
| decoder | string | `yolo_v8` | 后处理解码器类型：`yolo_v8`（默认）、`yolo_v5`、`yolo_x`（仅 detection/segmentation 有效） |

### 路径解析规则

- `model` 和 `labels` 字段支持相对路径和绝对路径
- 相对路径基于清单文件所在目录解析
- 路径统一通过 `QDir::cleanPath()` 规范化

## 任务类型说明

| task_type | 模型类 | 后处理 | 输出 |
| --- | --- | --- | --- |
| `detection` | YoloDetectionModel | 置信度过滤 + NMS 去重 + 坐标缩放 | 检测框 + 类别 + 置信度 |
| `classification` | ClassificationModel | softmax + Top-K 排序 | Top-K 类别 + 概率 |
| `segmentation` | SegmentationModel | 置信度过滤 + NMS 去重 + 掩码生成 | 检测框 + 掩码 + 类别 + 置信度 |

## 解码器说明

`decoder` 字段控制检测/分割模型的后处理策略，不同 YOLO 变体的输出张量格式不同：

| decoder | 适用模型 | 说明 |
| --- | --- | --- |
| `yolo_v8` | YOLOv8 / YOLOv8-seg | 默认解码器，输出形状 `[N, attrs]` |
| `yolo_v5` | YOLOv5 | 输出形状 `[1, attrs, N]` |
| `yolo_x` | YOLOX | 输出形状 `[1, attrs, N]` |

分类模型不使用 decoder 字段，后处理固定为 softmax + Top-K。

## 标签配置

标签有两种配置方式，优先级从高到低：

1. **labels_inline**：直接在 JSON 中以数组形式提供标签列表
2. **labels**：指定外部标签文件路径，每行一个标签

当 `labels_inline` 非空时，忽略 `labels` 字段。

### 标签文件格式

每行一个标签，空行自动跳过，行首尾空白自动去除：

```
person
bicycle
car
motorcycle
```

## ONNX 快速配置

当用户选择一个 ONNX 文件但尚无对应清单时，工具提供快速配置对话框：

1. 自动从 ONNX 文件名提取模型名称
2. 默认输入尺寸：640×640（步长 32，范围 32–4096）
3. 默认置信度阈值：0.25
4. 默认 NMS 阈值：0.45
5. 标签列表：可选，每行一个，自动去重和去除空白
6. 模型名称为必填项，空白时 OK 按钮禁用

配置完成后自动在 ONNX 文件同目录下生成 `{base_name}.json` 清单文件。

注意：直接导入 ONNX 文件时，当前只会生成 `detection` 类型的清单。`classification` / `segmentation` 类型请手动创建对应的 JSON 清单文件。

## 验证规则

### 清单文件验证

| 规则 | 说明 |
| --- | --- |
| name 非空 | `name` 字段必须为非空字符串 |
| task_type 合法 | 接受 `detection`、`classification`、`segmentation` |
| backend 合法 | 当前仅接受 `onnxruntime` |
| model 文件存在 | 解析后的模型文件路径必须存在 |
| input_width > 0 | 输入宽度必须为正整数 |
| input_height > 0 | 输入高度必须为正整数 |
| labels 文件存在 | 若配置了 `labels` 路径，文件必须存在 |

### ONNX 配置对话框验证

| 规则 | 说明 |
| --- | --- |
| 模型名称必填 | 去除空白后不能为空 |
| 输入尺寸范围 | 宽高范围 32–4096 |
| 输入尺寸步长 | 步长为 32 |
| 置信度阈值范围 | 0.0–1.0，步长 0.05 |
| NMS 阈值范围 | 0.0–1.0，步长 0.05 |
| 标签去重 | 相同标签只保留第一个 |
| 标签去空白 | 行首尾空白自动去除，空行跳过 |

## 适用场景

- 预配置的模型清单文件（如 `yolov8n.json`）
- 通过 ONNX 文件快速创建模型清单
- 批量管理多个模型的配置信息

## 注意事项

- 清单文件编码必须为 UTF-8
- JSON 格式必须合法，否则解析失败
- `model` 字段指向的 ONNX 文件必须在磁盘上存在
- 相对路径基于清单文件所在目录，移动清单文件后需检查路径有效性
- `labels_inline` 和 `labels` 可以同时存在，`labels_inline` 优先
