# 扩展推理后端（TensorRT / OpenVINO）

AIToolkit 通过 `BackendPlugin` + `BackendRegistry` 支持多推理后端。当前 **ONNX Runtime** 为唯一可用后端；TensorRT 与 OpenVINO 为预留插件。

## 架构

```
ModelManifest.backend  →  BackendRegistry.getBackend(name)
                              →  BackendPlugin::createModel()
                                   →  InferenceBackend (detect/classify/segment)
```

启动时在 `AppBootstrap::initialize()` 中注册：

- `registerOnnxRuntimePlugin()` — 可用
- `registerTensorRtPlugin()` — 骨架（`isAvailable=false`）
- `registerOpenVinoPlugin()` — 骨架（`isAvailable=false`）

## CMake 编译选项

| 选项 | 说明 |
|------|------|
| `-DAI_ENABLE_CUDA=ON` | ONNX Runtime CUDA Provider（需 `onnxruntime[cuda]`） |
| `-DAI_ENABLE_TENSORRT=ON` | 定义 `AI_TENSORRT_ENABLED`，为 TensorRT 集成预留 |
| `-DAI_ENABLE_OPENVINO=ON` | 定义 `AI_OPENVINO_ENABLED`，为 OpenVINO 集成预留 |

示例：

```bash
cmake --preset release -DAI_ENABLE_CUDA=ON -DAI_ENABLE_TENSORRT=ON
cmake --build build/release --config Release
```

> 启用 TensorRT/OpenVINO 选项后，插件会出现在「关于」与模型页后端列表中，但推理仍为占位实现，需链接对应 SDK 并完成 `createModel()` 实现。

## 实现新后端步骤

1. 在 `src/runtime/` 新增 `*_plugin.cpp`，实现 `BackendPlugin` 接口
2. 在 `app_bootstrap.cpp` 中调用 `register*Plugin()`
3. 模型清单 `backend` 字段指向插件 `info().name`（如 `onnxruntime`）
4. 补充 `tests/test_backend_registry.cpp` 与集成测试

## 模型清单字段

```json
{
  "name": "YOLOv8n",
  "task_type": "detection",
  "backend": "onnxruntime",
  "model": "yolov8n.onnx"
}
```

未来 TensorRT 引擎可设为 `"backend": "tensorrt"`，并增加 `engine` 或 `model` 路径字段（待规范扩展）。
